# ClickBook 全面コードレビュー — 2026/05/20

## 総合評価

全体的に高品質なプロジェクト。Chrome MV3 拡張のアーキテクチャ、React + Tailwind のUI実装、AI分類（Gemini Nano）統合、パターン管理・メモ機能など、多くの機能がまとまっている。以下は改善余地のある箇所を体系的に整理したもの。

## 1. バグ・不具合（要修正）

### 1-1. `BookmarkCard` のタイトルがダークモードでハードコード

**ファイル:** `src/components/BookmarkCard.tsx`

タイトルボタンのテキストカラーが `text-gray-200` 固定のため、ライトモードで背景色と同化して読みにくい。

```tsx
// 現在

// 修正案
className="text-left text-sm font-medium text-gray-800 dark:text-gray-200 hover:text-indigo-500 dark:hover:text-indigo-300 ..."
```

### 1-2. `RankingWidget` のテキスト色がダークモード固定

**ファイル:** `src/components/RankingWidget.tsx`

見出し `text-gray-300`、リンク `text-gray-300`、空メッセージ `text-gray-600` — すべてダークモード前提のカラー。ライトモードでコントラスト不足。

```tsx
// 修正案（各所）
"text-gray-700 dark:text-gray-300"  // 見出し・リンク
"text-gray-400 dark:text-gray-600"  // 空メッセージ
```

### 1-3. `RecentWidget` の見出し色もダークモード固定

**ファイル:** `src/components/RecentWidget.tsx`

```tsx
// 現在
<h2 className="text-sm font-semibold text-gray-300">最近追加</h2>

// 修正案
<h2 className="text-sm font-semibold text-gray-700 dark:text-gray-300">最近追加</h2>
```

同様に空メッセージの `text-gray-600` も要修正。

### 1-4. `useDialog` — confirm variant の分岐が同一

**ファイル:** `src/shared/useDialog.tsx`

```tsx
// 現在: warn と info で同じスタイル
type === "confirm"
  ? variant === "warn"
    ? "bg-indigo-600 hover:bg-indigo-500 text-white"   // ← warn
    : "bg-indigo-600 hover:bg-indigo-500 text-white"   // ← info（同じ）
```

`warn` の場合はユーザーに注意喚起するため `bg-red-600 hover:bg-red-500` 等にすべき。

### 1-5. `MessageResponse` 型に `isDuplicate` が不足

**ファイル:** `src/shared/types.ts`

`success: true` 側に `isDuplicate` がないが、service-worker は `{ success: false, ..., isDuplicate: true }` を返す。型定義は正しいが、Popup.tsx が `res.isDuplicate` をアクセスする際に型ガードなしで使っている箇所がある。

### 1-6. フォルダー削除に確認ダイアログがない

**ファイル:** `src/components/Sidebar.tsx` — `handleDelete`

ブックマークが含まれるフォルダーでも即削除される。ユーザーが誤操作で失うデータが大きいため、確認ダイアログを入れるべき。

```tsx
// 現在
async function handleDelete(id: string) {
  await chrome.runtime.sendMessage({ type: "DELETE_FOLDER", id });
  ...
}

// 修正案
async function handleDelete(id: string) {
  const count = counts[id] ?? 0;
  const msg = count > 0
    ? `このフォルダーには ${count} 件のサイトがあります。削除すると「その他」に移動されます。続けますか？`
    : `このフォルダーを削除しますか？`;
  if (!await showConfirm(msg, "削除する")) return;
  await chrome.runtime.sendMessage({ type: "DELETE_FOLDER", id });
  ...
}
```

## 2. パフォーマンス改善

### 2-1. storage の read-modify-write 競合リスク

**ファイル:** `src/shared/storage.ts`

すべての書き込み関数が `readStorage()` → 変更 → `writeStorage()` のパターン。短時間に複数操作が走ると、先の書き込みが後の読み取りに反映される前に上書きされる可能性がある（race condition）。

**対策案:**
- ロック機構（`navigator.locks.request()`）を導入
- または in-memory キャッシュを持ち、debounce してストレージに書き出す

```ts
async function withStorageLock<T>(fn: (data: StorageData) => Promise<T>): Promise<T> {
  return navigator.locks.request("clickbook_storage", async () => {
    const data = await readStorage();
    const result = await fn(data);
    return result;
  });
}
```

### 2-2. `bulkImport` で1件ずつ `addBookmark` を呼んでいる

**ファイル:** `src/background/service-worker.ts`

`addBookmark` は毎回 `readStorage()` + `writeStorage()` を実行するため、100件インポートすると200回のストレージI/Oが走る。

**対策案:** バルク書き込み関数を用意する。

```ts
export async function addBookmarks(newBookmarks: Bookmark[]): Promise<void> {
  const data = await readStorage();
  data.bookmarks = [...newBookmarks, ...data.bookmarks];
  await writeStorage(data);
}
```

### 2-3. `App.tsx` の `getBookmarksForFolder` が毎レンダーで計算

**対策案:** `useMemo` でメモ化する。

```tsx
const folderBookmarks = useMemo(
  () => selectedFolderId ? getBookmarksForFolder(selectedFolderId) : [],
  [selectedFolderId, filtered, folders]
);
```

## 3. セキュリティ

### 3-1. `importData` のバリデーション不足

**ファイル:** `src/shared/storage.ts`

`incoming.bookmarks` が配列かどうかだけチェックしているが、各要素の構造は検証していない。悪意あるJSONを食わせると不正なデータが保存される。

**対策案:** 各ブックマークの必須フィールド（id, url, title）の型チェックを追加。

```ts
function isValidBookmark(b: unknown): b is Bookmark {
  return typeof b === "object" && b !== null
    && typeof (b as Bookmark).id === "string"
    && typeof (b as Bookmark).url === "string"
    && typeof (b as Bookmark).title === "string";
}
```

### 3-2. HTML エクスポートの XSS

**ファイル:** `src/components/SearchBar.tsx` — `handleExportHTML`

```tsx
const rows = bookmarks
  .map((b) => `<li><a href="${b.url}">${b.title}</a></li>`)
  .join("\n");
```

タイトルやURLに `<script>` 等が含まれるとXSSの危険。HTMLエスケープを実施すべき。

```ts
function escapeHtml(s: string): string {
  return s.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;');
}
```

## 4. 機能改善・追加提案

### 4-1. ブックマーク開封時の `visitCount` が増えていない

`BookmarkCard` の `handleOpen` で `window.open()` しているだけ。`visitCount` を増やすロジックがどこにもない。ランキング機能が意味を持つには、クリック時にカウンターを更新する必要がある。

**対策案:** `handleOpen` 内で service-worker にメッセージを送信して `visitCount` をインクリメントする。

```ts
// types.ts に追加
| { type: "INCREMENT_VISIT"; id: string }

// service-worker.ts に追加
case "INCREMENT_VISIT": {
  const data = await readStorage();
  data.bookmarks = data.bookmarks.map(b =>
    b.id === message.id ? { ...b, visitCount: b.visitCount + 1 } : b
  );
  await writeStorage(data);
  return { success: true };
}
```

### 4-2. ソート機能がない

フォルダービューやダッシュボードで、ブックマークの並び順を変更できない。

**提案:** 名前順 / 追加日順 / 訪問回数順 のソートボタンを追加。

### 4-3. Popup のテーマが NewTab と独立

Popup で `localStorage` ベースでテーマを管理しているが、NewTab は `ThemeContext` + `localStorage` 。両方で `clickbook_theme` キーを使っているので同期はされるが、Popup では `ThemeProvider` を使っていないため `document.documentElement` に `dark` クラスが付与されない。

**対策案:** Popup にも `ThemeProvider` を適用する。もしくは `chrome.storage.local` でテーマを管理し、両環境で同期する。

### 4-4. キーボードショートカットがない

拡張として `commands` API を使い、ショートカットキーで「現在のタブを保存」ができると便利。

```json
// manifest.json に追加
"commands": {
  "save-current-tab": {
    "suggested_key": { "default": "Alt+S" },
    "description": "現在のタブをブックマークとして保存"
  }
}
```

### 4-5. 検索のデバウンスがない

**ファイル:** `src/newtab/App.tsx`

`searchQuery` の `onChange` が毎キーストロークで発火。データ量が多いと一瞬もたつく可能性がある。

**対策案:** `useDeferredValue` または `debounce` を導入。

### 4-6. ブックマーク削除時にメモが残る

`deleteBookmark` は `storage.ts` のブックマーク配列から削除するが、関連する `memos` は削除しない。孤児メモが溜まっていく。

**対策案:** `deleteBookmark` 時に対応メモも削除。

```ts
export async function deleteBookmark(id: string): Promise<void> {
  const data = await readStorage();
  data.bookmarks = data.bookmarks.filter((b) => b.id !== id);
  await writeStorage(data);
  // 関連メモも削除
  await deleteMemo(id);
}
```

## 5. コード品質

### 5-1. コード重複: `extractUrls` が2箇所に存在

`SearchBar.tsx` と `Popup.tsx` に同じロジックが重複。`shared/utils.ts` に共通化すべき。

### 5-2. コード重複: テキストインポートUI

`SearchBar.tsx` と `ChromeBookmarkPanel.tsx` と `Popup.tsx` に類似のテキストインポートUIが3箇所。コンポーネントとして切り出すと保守性が向上する。

### 5-3. カラー定数の重複

`MEMO_DOT`, `MEMO_TEXTAREA_BG`, `COLOR_DOT` 等が `BookmarkCard.tsx`, `MemoBoard.tsx`, `FolderView.tsx` に散在。`shared/colors.ts` に統合すべき。

### 5-4. `Popup.tsx` が肥大化

1つのコンポーネントで400行超。保存ロジック、一括保存、テキストインポート、コピーツール、Chrome連携、設定まで全部入り。関心ごとに分割を検討。

### 5-5. `service-worker.ts` の dynamic import 多用

`handleMessage` 内で多くの `case` が `await import(...)` を使っている。初回のメッセージでは問題ないが、パターンとして冗長。上部でまとめて import し、必要なもののみ使うのがシンプル。

## 6. UI/UX 改善提案

### 6-1. ライトモード対応の不完全さ（最重要）

上記のバグセクションで触れた通り、複数コンポーネントでダークモード固定のカラーが使われている。ライトモードで使用すると読みにくい箇所が多数。**全コンポーネントで `text-xxx dark:text-xxx` パターンの統一的なレビュー**が必要。

**該当コンポーネント:**
- `BookmarkCard.tsx` — タイトル、URL表示
- `RankingWidget.tsx` — 全体
- `RecentWidget.tsx` — 見出し、空メッセージ
- `ThemeToggle.tsx` — ホバー色 `hover:bg-surface-800` がライトモードで暗すぎる

### 6-2. 空状態（Empty State）の改善

フォルダーが空の場合 `📭 このフォルダーにはまだ何もありません` とだけ表示。CTA（Call To Action）を追加すべき。

```
📭 このフォルダーにはまだ何もありません
[+ サイトを追加]  [Chrome から取込む]
```

### 6-3. ダッシュボードのフォルダーカードで絵文字がハードコード

**ファイル:** `src/pages/Dashboard.tsx`

`EMOJI_MAP` で `technology: "💻"` 等を固定しているが、ユーザーがフォルダーにカスタム絵文字を設定していてもダッシュボードでは反映されない。`folder.icon` を直接使うべき。

```tsx
// 現在
<span className="text-lg">{EMOJI_MAP[f.id] ?? "📂"}</span>

// 修正案
<span className="text-lg">
  {isEmoji(f.icon) ? f.icon : (EMOJI_MAP[f.id] ?? "📂")}
</span>
```

### 6-4. サイドバーの幅が固定

`w-60` 固定で、長いフォルダー名が `truncate` で切れる。リサイズハンドルまたは最低限ワイド/ナロー切り替えがあると便利。

### 6-5. ブックマークカードにフォルダー名の表示がない

`BookmarkCard` にどのフォルダーに属しているかの表示がない。ダッシュボードの「最近追加」で表示されるカードは、どのフォルダーに分類されたのかが分からない。

**提案:** ドメイン表示の横にフォルダーバッジを小さく表示。

### 6-6. ドラッグ＆ドロップのフィードバックが弱い

ドラッグ中のプレビューがブラウザデフォルトのみ。カスタムドラッグプレビューやドロップターゲットのハイライトをもう少し明確にすると使いやすくなる。

### 6-7. モバイル（レスポンシブ）対応

New Tab ページは Popup と違いフルスクリーン表示だが、サイドバーが `w-60` 固定で小画面では使いにくい。Popup は `w-96` 固定で問題ないが、New Tab 側はブレークポイントでサイドバーを折り畳めるようにしたい。

### 6-8. Popup の「管理ページへ」ボタンの位置

現在コピーツールの上にあるが、ページ最下部よりも上部ヘッダー付近にあった方がアクセスしやすい。

### 6-9. ランキングパネルが App.tsx 内にインライン実装

App.tsx の中に170行以上のランキングUIがインラインで書かれている。`RankingWidget` は別ファイルにあるのに、サイドパネルのランキングは直書き。`RankingPanel` として分離し、スタイルも統一すべき。

### 6-10. トランジション・アニメーション

パネルの開閉（Chrome連携、編集、ランキング）は `transition-all duration-300` で幅が変化するが、中身がいきなり現れるため少しぎこちない。`opacity` と `translate` を組み合わせると滑らかに見える。

## 7. アーキテクチャ改善

### 7-1. 状態管理

現在は `App.tsx` が全状態を持ち、`loadData` を各コンポーネントに props で渡すパターン。機能が増えると props drilling が深くなる。`useContext` + `useReducer` でグローバルストアを構築するか、`zustand` 等の軽量ライブラリを検討。

### 7-2. メッセージ型の安全性

`chrome.runtime.sendMessage` の戻り値が `as MessageResponse` でキャスト。型安全にするには、メッセージタイプごとにレスポンス型を対応付けたユーティリティ関数を用意するとよい。

```ts
async function sendMsg<T extends Message>(msg: T): Promise<MessageResponse> {
  return chrome.runtime.sendMessage(msg) as Promise<MessageResponse>;
}
```

### 7-3. テストがない

ユニットテスト・E2Eテストが見当たらない。最低限、`categorizer.ts`、`storage.ts`、`buildFolderTree` 等の純粋ロジックにテストを追加すべき。
